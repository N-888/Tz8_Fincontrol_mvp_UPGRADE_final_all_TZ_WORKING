# Tz8_Fincontrol_mvp
FinControl — аккуратный и практичный MVP-сервис для учёта личных финансов с веб-интерфейсом на Django и Telegram-ботом для быстрого ввода операций и получения сводной аналитики. Проект задуман как чистый, расширяемый и понятный MVP, который можно использовать как учебную итоговую работу, демонстрационный прототип или основу для дальнейшего развития продукта.

# 1. Описание проекта

FinControl — это сервис для ведения личного финансового учёта, позволяющий пользователю:

- вручную добавлять доходы и расходы
- указывать категорию и описание операции
- видеть историю операций
- анализировать финансовую активность по периодам
- отслеживать баланс
- получать рекомендации по поведению расходов
- пользоваться Telegram-ботом как быстрым каналом доступа к данным

Система объединяет два канала взаимодействия:

## Веб-часть
Полноценный личный кабинет пользователя с историей, аналитикой, фильтрами и настройками.

## Telegram-бот
Быстрый интерфейс для:
- получения сводок
- сравнения периодов
- добавления доходов и расходов
- быстрого взаимодействия без входа на сайт

# 5. Архитектура решения

Проект построен по понятной и логичной архитектуре.

## Основные уровни

# 11. Активация окружения и установка зависимостей

## Если виртуальное окружение уже создано

Для PowerShell выполните:

```powershell
.\.venv\Scripts\Activate.ps1
```

## Если виртуального окружения ещё нет

Создайте его:

```powershell
py -3.14 -m venv .venv
```

Потом активируйте:

```powershell
.\.venv\Scripts\Activate.ps1
```

## Установите зависимости проекта

```powershell
pip install -r requirements.txt
```

# 12. Настройка переменных окружения

Создайте файл `.env` рядом с `.env.example`.

Пример содержимого файла `.env`:

```env
DJANGO_SECRET_KEY=my-super-secret-key-123456789
DJANGO_DEBUG=True
DJANGO_ALLOWED_HOSTS=127.0.0.1,localhost
TELEGRAM_BOT_TOKEN=ВАШ_ТОКЕН_ОТ_BOTFATHER
DJANGO_TIME_ZONE=Europe/Moscow
```

## Описание переменных

* `DJANGO_SECRET_KEY` — секретный ключ Django
* `DJANGO_DEBUG` — режим разработки
* `DJANGO_ALLOWED_HOSTS` — разрешённые хосты
* `TELEGRAM_BOT_TOKEN` — токен бота из BotFather
* `DJANGO_TIME_ZONE` — часовой пояс проекта

# 18. Правила и логика использования проекта

Для корректного и полезного использования системы рекомендуется:

* регулярно фиксировать все доходы
* регулярно фиксировать все расходы
* использовать корректные категории
* не пропускать даты операций
* проверять аналитику хотя бы раз в неделю
* следить за предупреждениями о лимитах, которые устанавливаются в веб-версии
* использовать Telegram для быстрых повседневных действий
* использовать веб-кабинет для анализа и контроля картины целиком

## Практический смысл для пользователя

Чем аккуратнее пользователь ведёт учёт:

* тем точнее становится аналитика
* тем понятнее структура его расходов
* тем раньше видны проблемные зоны
* тем легче выстраивать устойчивое финансовое поведение
